# core/fidelity.md
# zh-human-writing v1 — 保真机制

---

## 1. 核心原则

1. **不使用综合分**：每个检查项独立判定 pass/fail/warning
2. **分层检查**：确定性检查（fail → 回滚）和 warning 检查（进入待确认项）分开
3. **局部回滚**：硬约束失败时只回滚该处编辑
4. **不自动重试**：回滚后不自动重试

---

## 2. 可确定性检查（fail → 局部回滚）

| 检查项 | 检查方式 | 允许的变化 | 不允许的变化 |
|--------|---------|-----------|------------|
| 数字 | 逐字比对 | 千分位格式（1,000 ↔ 1000） | 数值变化 |
| 日期 | 逐字比对 | YYYY-MM-DD ↔ YYYY年M月D日 | 日期值变化 |
| 百分比 | 逐字比对 | 50% ↔ 50％ | 数值变化 |
| URL | 逐字比对 | http ↔ https（如原文已有） | URL 路径变化 |
| 代码块 | 逐字比对 | 无 | 代码内容任何变化 |
| 命令 | 逐字比对 | 无 | 命令任何变化 |
| 路径 | 逐字比对 | / ↔ \ | 路径内容变化 |
| 引用 | 逐字比对 | 无 | 引用原文任何变化 |
| 用户 protected spans | 逐字比对 | 无 | 任何变化 |

**检测者**：`scripts/fidelity_guard.py`

---

## 3. Warning 检查（进入待确认项）

| 检查项 | 说明 |
|--------|------|
| 否定变化 | 否定消失或反转 |
| 条件变化 | 条件消失、改变或增加 |
| 因果变化 | 因果消失、反转或改变 |
| 责任主体变化 | 观点归属转移、模糊或删除 |
| 不确定程度变化 | "可能" → "确定" 等 |
| 语体漂移 | 口语→书面 或 书面→口语 |
| 作者声音漂移 | 语气、立场、风格变化 |

**检测者**：LLM 语义检查（脚本只负责收集和格式化）

**注意**：profile 约束优先。technical 允许书面语，social 允许口语。作者声音漂移仅在 author_written 路径和有作者样本时检查。

---

## 4. 回滚条件

### 必须回滚

确定性检查 fail 时触发局部回滚：
1. 数字/日期/百分比/单位被改变
2. URL 被改变
3. 代码/命令/路径/参数/字段被改变
4. 日志/错误码/状态码被改变
5. 引用原文被改变
6. 用户显式 protected spans 被改变

### 不回滚但标记

warning 检查结果不回滚，但标记为待确认项：
1. 否定变化
2. 条件变化
3. 因果变化
4. 责任主体变化
5. 不确定程度变化
6. 语体漂移
7. 作者声音漂移

### 回滚后处理

1. 恢复到编辑前状态
2. 标注"此处编辑因保真约束失败已回滚"
3. 列入待确认项
4. 不自动重试

---

## 5. 双向保真审计

### 正向审计（原文 → 编辑后）

检查编辑后的文本是否保留了原文的所有 protected spans 和语义要素。

### 反向审计（编辑后 → 原文）

检查编辑后的文本是否引入了原文没有的内容（fabrication 检查）。

### 审计独立性

- 审计由独立步骤执行（步骤 6），不是改写步骤（步骤 5）的一部分
- 审计不改写文本，只检查
- 审计结果直接决定回滚和待确认项

---

## 6. 残留检查（步骤 8）

- 在步骤 7 完成后执行
- 只执行一次
- 只检查 hard-residue 是否残留
- 不检查 advisory-only 和 strong-contextual
- 不自动再次编辑
- 不进入循环

---

## 7. 不负责

- 不做评分
- 不做 AI 概率
- 不做质量评估
- 不做改写
- 不做自动重试
- 不做综合分计算
- 不做 MPS 计算
- 不做 fidelity score
