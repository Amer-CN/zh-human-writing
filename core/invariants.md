# core/invariants.md
# zh-human-writing v1 — 不可变约束
# 这些约束在所有 source、strategy、profile、length_retention 和 output 组合下均生效。

---

## 1. 必须保护的内容

### 1.1 事实性内容

| 类别 | 检查方式 | 违反后果 |
|------|---------|---------|
| 数字 | 逐字比对（允许千分位格式变化） | 局部回滚 |
| 日期 | 逐字比对（允许 YYYY-MM-DD ↔ YYYY年M月D日 格式变化） | 局部回滚 |
| 百分比 | 逐字比对 | 局部回滚 |
| 单位 | 逐字比对 | 局部回滚 |
| URL | 逐字比对 | 局部回滚 |
| 人名 | 逐字比对 | 局部回滚 |
| 机构名 | 逐字比对 | 局部回滚 |
| 产品名 | 逐字比对 | 局部回滚 |

### 1.2 技术性内容

| 类别 | 检查方式 | 违反后果 |
|------|---------|---------|
| 代码块 | 逐字比对 | 局部回滚 |
| 命令 | 逐字比对 | 局部回滚 |
| 路径 | 逐字比对 | 局部回滚 |
| 参数 | 逐字比对 | 局部回滚 |
| 字段名 | 逐字比对 | 局部回滚 |
| 日志 | 逐字比对 | 局部回滚 |
| 错误码 | 逐字比对 | 局部回滚 |
| 状态码 | 逐字比对 | 局部回滚 |

### 1.3 语义性内容

| 类别 | 检查方式 | 违反后果 |
|------|---------|---------|
| 引用原文 | 逐字比对 | 局部回滚 |
| 否定 | 语义比对（否定不能消失或反转） | warning + 待确认 |
| 条件 | 语义比对（条件不能消失或改变） | warning + 待确认 |
| 因果 | 语义比对（因果关系不能消失或反转） | warning + 待确认 |
| 责任主体 | 语义比对（观点归属不能转移） | warning + 待确认 |
| 不确定程度 | 语义比对（"可能"不能变成"确定"） | warning + 待确认 |
| 用户显式 protected spans | 逐字比对 | 局部回滚 |

### 1.4 作者性内容

| 类别 | 检查方式 | 违反后果 |
|------|---------|---------|
| 作者明确指定不可改的片段 | 逐字比对 | 局部回滚 |
| 作者的个人观点和立场 | 语义比对（不能被改变、软化或反转） | warning + 待确认 |
| 作者的语气和语域 | 语体比对（不能被统一为另一种腔调） | warning |

## 2. 禁止的动作

### 2.1 内容捏造类

- 无依据增加具体细节
- 无依据增加第一人称
- 无依据增加观点
- 无依据增加感受
- 无依据增加经历
- 自动补全模板占位值

### 2.2 伪装类

- 为了"人味"故意制造毛边
- 强制增加观点和第一人称
- 强制制造长短句
- 把所有文章改成同一种"人味腔"

### 2.3 评分类

- 用综合分抵消硬约束失败
- 使用数字评分作为自动循环门槛
- 输出 AI 概率
- 输出人味分
- 输出综合质量分

### 2.4 检测类

- 自动判断文本是不是 AI 写的
- 真人停手门检
- stylometry
- MATTR
- burstiness
- ouroboros

### 2.5 加载类

- 一次载入全部模式
- 固定大型行业白名单
- 全量复制外部 Skill 内容

## 3. 回滚规则

### 3.1 何时必须回滚

1. 数字/日期/百分比/单位被改变
2. URL 被改变
3. 代码/命令/路径/参数/字段被改变
4. 日志/错误码/状态码被改变
5. 引用原文被改变
6. 用户显式 protected spans 被改变
7. 否定消失或反转
8. 条件消失或改变
9. 因果关系消失或反转
10. 责任主体和观点归属被转移
11. 用户明确指定不可改的片段被修改

### 3.2 回滚范围

- **局部回滚**：只回滚违反约束的那一处编辑，不影响其他已通过的编辑。
- 不做全局回滚（除非全文保真审计发现系统性问题）。

### 3.3 回滚后处理

1. 标注"此处编辑因保真约束失败已回滚"
2. 在 diff/audit 输出中列入"待确认项"
3. 不自动重试该处编辑

---

## 4. 不使用综合分

- 每个检查项独立判定 pass/fail/warning，不汇总为综合分。
- 硬约束失败时回滚该处编辑，不做全局评分判断。
- 主观质量维度降级为 diff/audit 输出中的待确认项。

---

## 5. 执行流程约束

### 5.1 固定 9 步

1. 读取输入合同
2. 提取 protected spans
3. 识别 source、strategy 和 profile 对应权限
4. 诊断真实表达问题
5. 执行最小必要编辑
6. 独立做双向保真审计
7. 硬约束失败则局部回滚
8. 最多做一次高置信残留检查
9. 按 clean/diff/audit 输出

### 5.2 禁止的流程变体

- 不实现无限循环
- 不实现评分驱动迭代
- 不实现多轮改写（ouroboros）
- 不实现第二轮自由改写
- 残留检查最多一次，且只检查高置信模式

---

## 6. advisory-only 模式约束

advisory-only 模式的特征不得单独触发自动修改：

- "不是……而是……"
- "先……再……"
- "从……到……"
- 破折号
- 三段式
- 反问
- 二人称
- 句长相近
- 段长相近
- 普通连接词
- 第一人称
- 对仗和重复

这些模式只在 diff/audit 输出中作为建议列出，不自动改写。
