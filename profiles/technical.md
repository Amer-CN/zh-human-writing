# profiles/technical.md
# zh-human-writing v1 — technical profile

---

## 编辑目标

- 保证技术准确性
- 提升清晰度和可读性
- 保持步骤逻辑和参数完整性

---

## 重点保护项

| 保护项 | 说明 |
|--------|------|
| 代码块 | 逐字保护 |
| 命令和参数 | 逐字保护 |
| 路径 | 逐字保护 |
| 字段名 | 逐字保护 |
| API 名称 | 逐字保护 |
| 配置项 | 逐字保护 |
| 步骤顺序 | 不得重排 |
| 术语 | 不得替换为非标准译名 |
| 报错和日志 | 逐字保护 |
| 版本号 | 逐字保护 |

---

## 允许的语气

- 正式、精确的技术表达
- 被动语态（中文技术文档中的常态语法）
- 中英混排（技术文档的常态）
- 步骤说明（"先…再…"在技术文档中是合法的）
- 条件和假设语句

---

## 合法但容易误判的表达

| 表达 | 误判原因 | 正确处理 |
|------|---------|---------|
| "先配置 A，再配置 B" | 被当作阶段递进结构壳 | 技术步骤说明，保留 |
| "不是 A 而是 B"（精确对比） | 被当作二元对立 | 技术诊断中的精确对比是合法的 |
| 被动语态 | 被当作 AI 腔 | 中文技术文档中被动语态是常态 |
| 无主语句 | 被当作远距离叙述者 | 中文无主语句是常态语法 |
| 中英混排 | 被当作术语混乱 | 技术文档中英混排是常态 |
| "需要注意的是" | 被当作填充短语 | 如果后接重要警告，保留 |
| 列表和表格 | 被当作格式化 | 技术文档的列表和表格是必要的 |
| 长参数说明 | 被当作冗长 | 参数说明是必要的，不删 |

---

## 禁止自动执行的动作

- 替换技术术语为非标准译名
- 改变步骤顺序
- 删除代码注释
- 修改命令和参数
- 合并步骤
- 拆分代码块
- 删除错误处理说明
- 删除性能数据
- 改变版本号或日期
- 把中英混排统一为纯中文

---

## 默认 strategy

由 source 推导：
- author_written → preserve
- ai_draft → balance
- mixed → balance
- unknown → preserve

## 加载的模式包

- `references/patterns/hard-residue.md`（诊断发现时）
- `references/patterns/strong-contextual.md`（诊断发现时，阈值放宽 ×1.5）
- `references/patterns/advisory-only.md`（output=diff 或 audit 时）
- `references/protected-spans.md`（始终加载）
- `references/false-positive-protection.md`（始终加载）

---

## 默认策略

- 导航和总结的阈值放宽（×1.5）
- 步骤说明不当作阶段递进结构壳
- 被动语态不当作 AI 腔
- 中英混排不当作术语混乱
